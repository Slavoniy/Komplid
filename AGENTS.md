# Instructions for AI Agent (Jules)

## О проекте
Полностековый веб-сервис для строительной отрасли. Главная бизнес-цель — автоматизация создания Актов освидетельствования скрытых работ (АОСР). Система парсит строительные сметы, анализирует данные с помощью LLM (Gemini) и генерирует готовые документы в формате Word на основе заранее подготовленных шаблонов.

## Управление задачами (Task Management) - КРИТИЧЕСКИ ВАЖНО
* **Единый источник истины:** В корне проекта лежит файл `todo.md`. Перед началом любых действий ты обязан прочитать его текущее состояние.
* **Фокус:** Бери в работу строго **одну задачу** (один чекбокс `[ ]`) за раз из раздела "Текущий спринт". Не пытайся реализовать весь функционал сразу.
* **Обновление статуса:** Как только ты завершил логический этап и убедился, что код работает, самостоятельно измени статус этой задачи в `todo.md` на выполненный `[x]`. 
* **Отслеживание багов:** Если в процессе работы ты замечаешь баги, которые не относятся к текущей задаче, не отвлекайся на них. Просто допиши их в раздел "Баги и исправления (Backlog)" файла `todo.md`.

## Технологический стек
* **Frontend:** React 19, Vite, TailwindCSS v4, иконки Lucide React.
* **Backend:** Node.js, Express.js (всё инициализируется в `server.ts`).
* **База данных:** PostgreSQL (через пакет `pg`). Важно: реализован in-memory фолбек (`memoryUsers`, `memoryProjects`) для работы приложения, если подключение к БД отсутствует.
* **Аутентификация:** JWT-токены (хранятся в `cookies`), хеширование паролей через `bcryptjs`.
* **Документы и файлы:** `docxtemplater` + `pizzip` (для генерации `.docx`), `xlsx` (для чтения смет).
* **AI Интеграция:** `@google/genai` (Gemini).

## Архитектура и запуск
* Приложение использует единый сервер (`server.ts`). В режиме разработки (dev) Express подключает Vite в режиме middleware для отдачи фронтенда. В продакшене (prod) Express отдает собранную статику из папки `dist`.
* **Запуск проекта:** `npm run dev` (используется `tsx server.ts`).
* **Сборка проекта:** `npm run build`.
* **API маршрутизация:** Все эндпоинты бэкенда находятся под префиксом `/api/` (например, `/api/auth/login`, `/api/projects`). Маршруты защищены middleware `authenticateToken`.

## Правила написания кода
1. Никогда не создавайте компонент длиннее 150 строк кода. Если его длина превышает это значение, автоматически разбейте его на более мелкие компоненты. Всегда разделяйте пользовательский интерфейс и логику.
2. **Не трогай настройки HMR в Vite:** В файле `vite.config.ts` HMR отключен для AI Studio (`hmr: process.env.DISABLE_HMR !== 'true'`). Это предотвращает мерцание экрана при редактировании кода агентом. Не изменяй эту логику.
3. **Работа с переменными окружения:** Ключи (`GEMINI_API_KEY`, `DATABASE_URL`, `JWT_SECRET`) берутся из `.env` или '.env.example'. **Никогда не хардкодь секреты.**
4. **Генерация документов:** При работе с созданием файлов Word всегда используй связку `docxtemplater` и `pizzip`. Ожидается, что шаблоны (например, `aosr_template.docx`) лежат в доступной для чтения директории.
5. **Обработка базы данных:** При добавлении новых маршрутов или функций для работы с данными, всегда пиши код так, чтобы он поддерживал два режима: работу с пулом PostgreSQL (`if (pool)`) и работу с In-Memory массивами (`else`).
6. **Стилизация:** Для фронтенда используй исключительно утилитарные классы TailwindCSS. Сторонние CSS-файлы создавать не нужно, за исключением базового `index.css`.
