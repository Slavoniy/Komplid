# Instructions for AI Agent (Jules)

# Описание проекта
B2B SaaS платформа для автоматизации строительной исполнительной документации на рынке РФ. Основной фокус: генерация АОСР, актов КС-2/КС-3 и ведение общих журналов работ.

# Технологический стек
- Backend: Python (FastAPI)
- Frontend: React + Tailwind CSS
- База данных: PostgreSQL
- Инфраструктура: Docker, Docker Compose

# Архитектурные правила и специфика предметной области
1. Специфика РФ: Система должна быть готова к развертыванию на серверах в РФ (ФЗ-152 о персональных данных). 
2. Документооборот: Все экспортируемые документы должны строго соответствовать формам Ростехнадзора, СНиП, СП и ГОСТ. Для генерации используй библиотеки `python-docx` и `reportlab` (или аналоги).
3. Качество кода: Используй строгую типизацию (Pydantic/MyPy для Python, TypeScript для React). 
4. API: Взаимодействие между фронтендом и бэкендом происходит строго через REST API (возвращай JSON).
5. Язык: Все комментарии в коде, названия коммитов и документация должны быть на русском языке.

# Правила написания кода
1. Никогда не создавайте компонент длиннее 150 строк кода. Если его длина превышает это значение, автоматически разбейте его на более мелкие компоненты. Всегда разделяйте пользовательский интерфейс и логику.
